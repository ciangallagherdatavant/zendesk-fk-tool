# Datavant Readability Tool — How to Use

## What This Tool Does
Automatically analyses Zendesk help centre articles for readability using
the Flesch-Kincaid Grade Level score (calculated by Python) and provides
WCAG 2.2 writing accessibility notes (generated by Claude AI).

The tool runs every 4 hours on weekdays (and 7am on weekends) — no
manual analysis needed.

**Live Dashboard:** https://ciangallagherdatavant.github.io/zendesk-fk-tool/

---

## How to Add a New Article for Analysis

### Step 1 — Copy the Zendesk URL
Open the article in Zendesk and copy the full URL from your browser.

### Step 2 — Open the Google Sheet
Click the "📊 Open Datavant Help Center Articles" button on the
dashboard, or go directly to the sheet linked in the dashboard.

### Step 3 — Paste the URL
- Paste the URL into **column A** of any new row
- Leave **column C (Status)** blank
- Save the sheet (it auto-saves)

### Step 4 — Wait or Trigger Manually
The tool runs automatically every 4 hours. To run it immediately:
1. Go to GitHub → Actions tab → "FK Readability Analysis"
2. Click "Run workflow" → "Run workflow"
3. Wait ~30 seconds

The dashboard will update within 1-2 minutes after the workflow completes.

## Reading the Dashboard

### Score Guide
| Score | Status | Meaning |
|---|---|---|
| 12.0 or below | ✅ Meets target | Accessible to most users |
| 12.1 to 14.9 | ⚠️ Needs improvement | Slightly too complex |
| 15.0 and above | 🔴 Needs significant work | Major rewrites required |

**Target:** Grade 12 or below for all Datavant help centre articles.

### Each Article Card Shows
- The current FK score and reading level
- A plain-English summary of what the score means
- Three specific readability recommendations
- WCAG 2.2 writing accessibility notes

### Sidebar Navigation
- **Overview** — scroll to top
- **Articles** — jump to article list
- **Run history** — see how scores have changed over time

## Architecture

This tool is a hybrid Python + Claude system:

- **Python (`fk_analysis.py`)** — fetches Zendesk articles, calculates
  the FK score using the `textstat` library (mathematical, consistent)
- **Claude AI** — generates plain-English explanations and WCAG notes
- **Google Sheets** — queue of articles to analyse
- **GitHub Actions** — runs the script automatically on schedule
- **GitHub Pages** — hosts the live dashboard

## Required Credentials (Stored as GitHub Secrets)

These must remain valid for the tool to keep working:

- `ANTHROPIC_API_KEY` — Claude API
- `ZENDESK_SUBDOMAIN`, `ZENDESK_EMAIL`, `ZENDESK_API_TOKEN` — Zendesk API
- `GOOGLE_SHEET_ID`, `GOOGLE_SHEETS_CREDENTIALS` — Sheets API service account

To rotate or update: Repo → Settings → Secrets and variables → Actions

## Troubleshooting

**Article shows "Re-run this article to generate recommendations"**
The result file exists but recommendations didn't parse. Re-add the
URL to the Google Sheet to re-run the analysis.

**Dashboard hasn't updated**
1. Check GitHub → Actions tab. Is the latest run green?
2. If red, click into it to see the error
3. If green, hard-refresh the dashboard (Cmd+Shift+R / Ctrl+Shift+R)

**API key expired**
Update the relevant secret in GitHub repo Settings → Secrets.
